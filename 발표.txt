안녕하세요 이번에 발표를 하게된 오근민이라고 합니다.

먼저 개발환경부터 확인하도록하겠습니다.
개발툴은 이클립스 4.21.0이고 윈도우시스템에서 작업했습니다.
프레임워크는 Spring boot2.6.2 이고 AWS RDS(마리아 DB)기반으로 작성했습니다.

가장먼저 프로젝트는 각 권한별 기능을 주기위하여 
- Spring Security 기반으로 만들었으며 
HttpSecurity Configure 를 통해 URL 접근 권한 관리 설정을하였으며
유저 권한에 따라 접근할 수 있는 페이지 따로 부여했습니다.
모두가 접근할수없는 페이지는 게시판 으로 적용했으며 인증이필요한경우 로그인페이지로 이동하여 인증후 사용가능하도록 설계를했습니다.

기본적으로 기능을 구현을 하기위해서는 인증 후 사용이 가능하기 때문에 회원가입 부터 
적용을 시켰습니다.
회원가입시 ID는 Email로 표현 했으며 해당형식 자바스크립트를 이용해 각각 이메일 비밀번호 규칙을 적용하였고 전체적으로 공백이 없게 설정을하였으며
만약 해당 유효성검사 수칙에 위배될경우 해당내용에관한 가이드가 나오도록 aleat으로 설정을 해놓았습니다.
기본적으로 회원가입을 하게되면 두개의 권한이 주어지는 ADMIN USER 두개의 권한을 갖는 ROLE을 설정하였으며 기본적으로 가입을할경우 USER권한이 부여되도록 설정하였습니다.
가입후 로그인시 Spring Security 를 통한 보안 로그인베이스로 구성을 하였으며 이메일이나 비밀번호가 일치하지않을경우 param.error 메시지가 출력되도록 설정하였으며
DB와 내용이 일치시 정상로그인 되도록 구현하였으며 OAuth2UserRequest를 구성하여 소셜아이디로도 인증이 가능하도록 구현을 하였습니다.

자유게시판의 경우 악성글혹은 댓글이 있는것을 대비하여 게시판과 댓글 Entity에 원투메니 메니투원을 설정하였고 케이스케이드를 이용하여 FK갑과 상관없이 삭제 가증하도록 구성하엿습니다.

자유게시판의 경우 내용확인은 누구나 확인 가능하게 설정하였으나 글쓰기 혹은 답글의 경우 회원인증 후  작성가능하도록 설정하였습니다.
